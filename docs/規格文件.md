# 自留保費統計表報表轉換系統 — 規格文件 v2

> **v2 更新說明**：依據回覆文件整合所有確認事項與變更需求。

## 專案目標

將多個產險公司的「自留保費統計表」Excel 來源檔，自動彙整填入輸出模板，產生季度/累計報表。

- **輸入**：多個 `公司別_年度(月-月)_自留保費統計表.xlsx`
- **輸出**：`{年度}年產險業務(Q{季}季自留)保費統計表.xlsx`

### v2 關鍵變更摘要

| # | 變更項目 | 原規格 | 新規格 |
|---|----------|--------|--------|
| 1 | 輸出格式 | `.xls` | **`.xlsx`** |
| 2 | 模板格式 | `.xls` | **`.xlsx`**（需先將範例模板轉檔） |
| 3 | 公式處理 | 全部寫值 | **保留 Excel 公式** |
| 4 | 公司清單 | 固定 19 家 | **動態**（由來源檔決定） |
| 5 | Sheet 讀取 | 按名稱匹配 | **只讀第一個 Sheet** |
| 6 | 多季度處理 | 一次一季 | **支援同時產出多季** |
| 7 | 技術選型 | xlrd + xlwt | **全部使用 openpyxl** |

## 3. 檔名解析規則

### Regex
```
^(\d{1,2})_(\d{2,3})\((\d{2})-(\d{2})\)_自留保費統計表\.xlsx$
```

### 欄位拆解

| 群組 | 名稱 | 範例值 | 說明 |
|------|------|--------|------|
| `\1` | 公司別 | `29` | 公司代號（1-2 位數字） |
| `\2` | 年度 | `115` | 民國年（2-3 位數字） |
| `\3` | 起始月 | `01` | 固定 2 位數 |
| `\4` | 結束月 | `03` | 固定 2 位數 |

### 錯誤格式處理策略

| 錯誤類型 | 處理方式 |
|----------|----------|
| 檔名不符 Regex | **跳過**此檔，記錄警告 |
| 月份不合理（>12 或 起始>結束） | **中止**此檔 |
| 副檔名非 `.xlsx` | **跳過** |

## 6. 資料整合策略

### 整合規則

| 項目 | 策略 |
|------|------|
| 多檔整合方式 | **分公司填入**：每個來源檔填入模板中對應公司的列 |
| 同欄位如何累加 | **不累加**：每家公司有獨立的列 |
| 缺值處理 | 若某公司無來源檔，該公司列保持模板原值（0 或空） |
| 重複資料處理 | 同一公司同一季度出現多個來源檔 → **中止並報錯** |
| 多季度同時 | 按各檔案的月份區間分組，填入對應季度區塊 |

### 多季度處理流程

```
1. 掃描所有來源檔
2. 按 (年度, 結束月) 分組 → 對應季度
3. 每個季度獨立填入對應區塊
4. 產出一份包含所有有資料季度的報表
```

| Sheet2 T 欄 | SUM(D:S) | **保留**，不寫入 |
| Sheet2 V 欄 | `(T/U)-1` | **保留**，不寫入 |
| Sheet2 小計列 | SUM(各公司同欄) | **保留**，不寫入 |
| Sheet2 總計列 | SUM(各季小計) | **保留**，不寫入 |

### 技術可行性

使用 `openpyxl` 操作 `.xlsx` 時：
- 讀取模板時**不使用** `data_only=True`（保留公式）
- 只對資料儲存格 `.value` 賦值
- 公式儲存格不觸碰 → 自然保留

### 風險

| 風險 | 緩解 |
